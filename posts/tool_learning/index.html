1.Git简介

版本控制是一种记录一个或若干文件内容变化，以便将来查阅特定版本修订情况的系统，方便查看更改历史，备份以及恢复以前的版本，保证多人的协作不出问题。

Git是一个开源的分布式版本控制工具。

1.版本控制工具的发展历史经过: 原始人工维护状态，本地RCS，集中式如VCS、SVN和分布式如Git。

2.版本控制工具提供了协作开发的能力，借助它们我们可以回到任何时间的代码状态。

3.集中式版本控制工具，几乎所有的动作都需要服务器参与，并且数据安全性与服务器关系很大。

4.Git 是分布式版本控制工具，除了与服务器之前进行按需同步之外，所有的提交操作都不需要服务器。

2.Git常用命令

  • 工程准备
    • git init
    • git clone
  • 查看工作区
    • git diff
    • git status
  • 文件修改后提交推送
    • 工作区 git add/ git rm/ git mv
    • 本地仓库 git commit
    • 远端仓库 git push
  • 查看日志
    • git log
  • 分支管理
    • 列出 git branch
    • 新建 git branch/ git checkout -b
    • 删除 git branch -d
    • 切换 git checkout
    • 更新 git pull
    • 合并 git merge
  • 撤销操作
    • git reset
    • git checkout
  • 分支合并
    • git merge/ git rebase
